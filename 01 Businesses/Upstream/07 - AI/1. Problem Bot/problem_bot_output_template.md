## Quality Criteria

Before generating the output, verify:

1. **The core problem is in buyer language.** Read it out loud. Does it sound like something a real person would say, or something a consultant would put on a slide? If the latter, rewrite.

2. **The core problem is specific enough to exclude people.** If everyone in the world has this problem, it's too broad. It should clearly apply to the ICP and not to everyone.

3. **The core problem is NOT a solution in disguise.** "Companies need better X" is a solution. "Companies keep losing their best people because X" is a problem. Check this explicitly.

4. **The language bank uses buyer words only.** Scan every phrase. If it sounds polished or professional, it's probably consultant language. Replace with the raw, unfiltered version.

5. **The Google searches are realistic.** These should be things a real person would actually type, not SEO keywords. "Why do my best employees keep quitting" is real. "Employee retention optimization strategies" is not.

6. **Current alternatives include "do nothing."** Inaction is almost always the biggest competitor. It must be listed with a clear reason why it fails.

7. **The Dinner Party Test assessment is honest.** If the problem statement wouldn't make someone lean in, say so and explain what needs to change. Don't let a weak statement through with a passing grade.

8. **Deprioritized problems have clear rationale.** The client should understand WHY these were set aside, not just that they were.
